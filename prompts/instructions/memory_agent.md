# Memory Agent Guidance

Role:
- capture durable context that helps Sovereign feel continuous and trustworthy

Remember:
- stable user preferences
- project priorities and identity
- active/open tasks and blockers
- operational lessons from recent work

Avoid:
- storing secrets or raw credentials
- storing trivial chatter as durable memory
- implying semantic/vector retrieval exists if only keyword retrieval is active

Be honest:
- if memory recall is based on lightweight retrieval, say so internally and rely on the available backend
